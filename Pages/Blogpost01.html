blog post 01 | some handy gitbash commands

by me

It's hard to remember these when you haven't used gitbash for a while. When I opened it up to start this project, I realised I'd forgotten them all. So here's a little list of the ones I had go digging through my memory (and goggle) to find!

change directory

Let's start with basics.

cd [foldername]

This one stands for change directory. By change it means open and by directory it means folder. Type in cd documents to move into the documents folder. You can also hit

cd ..

to step back one folder. And just

cd

to go all the way out. Hint: you can type a few letters of the folder you want and hit tab, it'll prefill the rest.

list files

This one means to list files. Looking for a folder? Can't work out if you're in the right spot? Can't remember the name of the folder you're looking for? You want this one:

ls

make directory

We're making a new folder, we're using gitbash, we want

mkdir [foldername]

Remember to leave out spaces or use ' ' (or " ") around your text if you want a foldername with spaces.

remove a file or directory

We've made it but we don't want it. You want:

rmdir [foldername]

And if it's a file you want to get rid of, it's just:

rm [filename]

that's all folks!
